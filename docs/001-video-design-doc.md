**日期**：2026-03-16

**分辨率**：1920×1080
**语言**：中英双语字幕
**配音**：无配音版（画面先行，配音后期另处理）
**背景音乐**：预留音轨，后期加入

## 一、视觉系统

### 色彩

| 角色       | 色值       | 用途                         |
|------------|------------|------------------------------|
| 背景主色   | `#0D0D14`  | 全局深色背景                 |
| 背景次色   | `#13131F`  | 卡片/面板背景                |
| 品牌主色   | `#1A6EFF`  | 主要信息、图标、线条         |
| 强调色     | `#FF6B1A`  | 关键数字、CTA、重点文字高亮  |
| 辅助青色   | `#00C4FF`  | 数据流动、次级强调           |
| 文字主色   | `#E8EDF5`  | 正文文字                     |
| 文字次色   | `#7A8599`  | 副标题、说明文字             |
| 分割线     | `#1E2235`  | 边框、分割                   |

### 字体

- 中文：系统无衬线黑体（`"PingFang SC", "Microsoft YaHei", sans-serif`）
- 英文/数字：`"JetBrains Mono"` 或 `"Inter"` 等宽/几何无衬线
- 字号层级：
  - 主标题：72px Bold
  - 副标题：40px Medium
  - 正文说明：28px Regular
  - 字幕：24px Regular，底部居中，半透明黑底

### 动画原则

- 缓动：`easeInOutCubic` 为主，数字计数用 `easeOut`
- 过渡：淡入 + 轻微上移（Y +20px → 0），时长 400-600ms
- 数字：计数器动画，从 0 增长到目标值
- 不用 3D 旋转、粒子爆炸等花哨效果
- 切场：硬切或 300ms 交叉淡变，不用推拉

### 段落 1：开场钩子（0 – 360f）

**叙事目标**：让目标用户（工厂管理者）立刻感到"这说的就是我的问题"。

**画面内容**：

三个问题卡片依次出现，每张停留约 3s：

1. `人工抄表，每天 2 小时`
   `— 还经常出错`

2. `设备停机，找原因靠经验`
   `— 数据没有留存`

3. `接个设备，要等厂商工程师`
   `— 少则一周，多则一月`

**视觉**：深色背景，文字逐行出现，强调词用橙色 `#FF6B1A`，每段末尾加细线下划。

**字幕（中/英）**：
- `人工抄表，每天耗时 2 小时，还经常出错`
- `Manual data logging: 2 hours daily, error-prone`

---

### 段落 2：品牌出场（360 – 660f）

**叙事目标**：清晰亮出产品身份与核心价值主张。

**画面内容**：

1. Logo 从中心放大出现（scale 0.6 → 1.0，400ms）
2. 品牌名 `蜻蜓助手` 从右侧滑入
3. slogan 渐入：`工业数据，快装快采快用`
4. 三个关键词 `快装 · 快采 · 快用` 单独放大强调，配橙色点

**字幕**：
- `蜻蜓助手——工业数据采集平台`
- `Autopaddle Assistant — Industrial Data Platform`

---

### 段落 3：快装（660 – 1860f，40s）

**叙事目标**：打消"部署复杂"的顾虑，强调零门槛。

**时间分配**：
- 0-8s：标题卡 `快装` 出场，副标题"办公电脑即可，无需专用硬件"
- 8-28s：三步流程动画（步骤连接线动画）
  - Step 1：下载安装包 → 双击安装（图标动画）
  - Step 2：配置内网地址（UI 截图：安装引导界面）
  - Step 3：设备自动发现，进入主界面（UI 截图：设备列表）
- 28-36s：强调卡 `数据留存内网，企业自主掌控`
- 36-40s：过渡到快采

**所需素材**：
- [ ] 安装引导界面截图
- [ ] 设备发现/列表界面截图

**字幕**：
- `办公电脑安装即用，无需专用服务器`
- `Runs on any office PC. No dedicated hardware required.`

---

### 段落 4：快采（1860 – 3510f，55s）

**叙事目标**：展示接入能力强、AI 降低门槛的核心差异点。

**时间分配**：
- 0-10s：标题卡 `快采` + `90% 数采场景，自助上线`（数字用橙色）
- 10-25s：AI 引导配置演示（UI 截图：选设备 → AI 推荐配置 → 一键接入）
- 25-50s：三种采集方式，各约 8s
  - 设备直连：局域网设备 + 协议图标动画（Modbus / OPC-UA / MQTT）
  - 视觉识别：摄像头 → 屏幕截图 → 数据提取 动画
  - 文件采集：文件图标 → 保存 → 自动同步 动画
- 50-55s：过渡到快用

**所需素材**：
- [ ] AI 引导配置界面截图

- [ ] 协议图标（可程序绘制）

**字幕**：
- `预置万种设备配置，AI 引导，跟着做就能上线`
- `AI-guided setup. 10,000+ device templates. 90% of cases need no expert.`

---

### 段落 5：快用（3510 – 4710f，40s）

**叙事目标**：展示数据变价值的最后一公里，"采到即用"。

**时间分配**：
- 0-8s：标题卡 `快用` + `一句话，生成 AI 应用`
- 8-22s：输入框动画：用户输入 `"按机床统计今日稼动率，异常自动报警"`

  - AI 分析中（加载动画 1s）
  - 生成设计方案卡片渐入
- 22-35s：UI 截图展示生成的看板/报表界面，数据图表动画渐入
- 35-40s：强调 `PC / 移动端同步查看`，手机+电脑双端图示

### 段落 6：数字收割（4710 – 5100f，13s）

**叙事目标**：用具体数字强化可信度，给销售对话提供弹药。

**画面**：四组数据卡片依次出现，每组约 3s：

| 数字       | 说明                           |
|------------|--------------------------------|
| `1 周`     | 完成基本接入，快速上线         |
| `90%`      | 数采场景可自助完成，无需专家   |
| `60%+`     | 故障发现时间缩短               |
| `2 小时/天`| 人工统计时间节省               |

数字用计数器动画 + 橙色大字，说明文字灰色小字。

## 三、技术实现规划

### 组件结构

```
src/
  compositions/
    QingTingVideo.tsx        # 主合成，负责整体时间轴
  scenes/
    01-OpeningHook.tsx
    02-BrandIntro.tsx
    03-QuickInstall.tsx
    04-QuickCollect.tsx
    05-QuickUse.tsx
    06-MetricsHighlight.tsx
    07-BrandOutro.tsx
  components/
    Subtitle.tsx             # 中英双语字幕
    TitleCard.tsx            # 段落标题卡
    MetricCard.tsx           # 数字强调卡
    StepFlow.tsx             # 步骤流程动画
    TypewriterText.tsx       # 打字机效果
    UIScreenshot.tsx         # 产品截图展示（带动画）
    CountUp.tsx              # 计数器动画
  constants/
    colors.ts                # 色彩系统
    timing.ts                # 时间常量（各场景帧起止）
  assets/
    screenshots/             # 产品 UI 截图（待提供）
    logo/                    # Logo 文件（待提供）
```

```ts
durationInFrames: 5400   // 180s × 30fps
fps: 30
width: 1920
height: 1080
```
